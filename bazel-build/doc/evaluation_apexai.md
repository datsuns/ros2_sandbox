# ROS 2 Bazel Build Evaluation: ApexAI/rules_ros

## 評価結果: 継続調査・時期尚早 (Low Priority)

ドキュメントにある通り "Work in Progress" の側面が強く、標準的な ROS 2 パッケージの多くが未だ Bazel 対応されていない。

### 1. ビルドの成功状況
- **diag_sample_node**: 失敗 (パッケージ不足)
- **pcl_sample_node**: 失敗 (パッケージ不足)

### 2. 技術的特徴
- **WORKSPACE 形式**: 従来の `WORKSPACE` ファイルによる依存管理が主体。
- **厳格な構成**: 独自の Python インタープリタ構成や、特定のリポジトリ名 (`@python_interpreter`) を要求する。
- **インジェクション方式**: ROS 2 のソースに外部から `BUILD` ファイルを注入する方式。

### 3. 直面した課題
- **対応パッケージの少なさ**: 
  - `diagnostic_updater` や `pcl_conversions` 等、ROS 2 の主要なパッケージが `configure_ros2` で生成されるリポジトリ群に含まれていない。
  - `bazel.repos` を確認したところ、多くのパッケージがコメントアウトされており、未実装の状態。
- **rootユーザー制限**: 
  - `rules_python` の制限により root での実行が困難（非rootユーザーへの移行で解決可能だが、構成が複雑）。
- **依存関係の記述**: 
  - `@ros2//:ros2` のような一括参照はできず、個別のレポジトリ名を知る必要がある（例: `@ros2.rclcpp//rclcpp`）。

### 4. 制限事項
- 独自のパッケージを追加する際の手順が確立されておらず、`rules_ros` 自体の改造が必要になる場面が多い。
- サンプルコードが非常に限られている。

### 5. 総合評価
Apex.AI 社によるプロジェクトであり、将来性は期待できるものの、現時点での汎用性は低い。特定の環境（例: QNX や極限まで制限された環境）での利用を想定している可能性が高い。
